OSA(Final Project)

1. Go to https://archives.fedoraproject.org/pub/archive/fedora/linux/releases/28/Workstation/x86_64/iso/

Download the fedora workstation iso file

2. Open your preferred virtualization software(I am using VMware workstation PRO)

Follow the steps in the video and create a virtual machine:

Create a new virtual machine with the following configurations:

  • Name: Centos Linux
  • Type: Linux
  • Version: CentOS 7/8 (64-bit)
  • Memory: 4096 MB (4 GB)
  • Hard Disk: Create a virtual hard disk now, 50GB, dynamically allocated.
  • We also attached the fedora 28 ISO image to the empty DVD drive.

3. After that install CentOS Stream 9 by following these steps:

  1. Start the virtual machine.
  1. The virtual machine will boot from the Fedora 28 ISO.
  1. Select "Install CentOS 9 Stream" and follow the on-screen instructions.
  1. Use Automatic partitioning for storage configuration.
  1. Set User and root password to 123456.
  1. Complete the installation and update CentOS 9 Stream Linux.

4. Now we are going to update our CentOS 9 to the latest version:

  • After the installation, boot into CentOS 9 Stream.
  • Open the terminal.
  • Run the following commands to update the system:
    [harshdhami@localhost ~]$ sudo dnf update -y

5. Now we are going to add a second hard drive to the virtual machine:

  • Power off the virtual machine.
  • Go to the settings of the virtual machine.
  • Navigate to the Storage section and add a new virtual hard disk of 10GB.

6. Now we are going to partition the new drive:

  • We are going to use the sudo fdisk -l command to list all the drives and then we will select our newly added harddrive:

From the output we can see that /dev/nvme0n2 is the newly added harddrive now we can work on it

  • We are going to use parted command for doing the partitions:
[harshdhami@localhost ~]$ sudo parted /dev/nvme0n2 mklabel gpt
[sudo] password for harshdhami: 
Information: You may need to update /etc/fstab.

This command will create a gpt partition table for our drive we can also use the older mbr table using msdos

  • Doing the partitions
[harshdhami@localhost ~]$ sudo parted /dev/nvme0n2 mkpart primary ext4 0% 2GB

[harshdhami@localhost ~]$ sudo parted /dev/nvme0n2 mkpart extended 2GB 10GB

10. Allow the system to Mount /dev/nvme0n2p3 Automatically at Boot:

harshdhami@localhost ~]$ sudo blkid /dev/nvme0n2p3
/dev/nvme0n2p3: UUID="[4cdfa328-1f6b-4d41-ba67-99cb99394b1e]" TYPE="ext2"

Add the entry to /etc/fstab:

UUID=[4cdfa328-1f6b-4d41-ba67-99cb99394b1e] /dev/nvme0n2p3 ext2 defaults 0 2

11. Prepare and Activate /dev/nvme0n2p4 as Virtual Memory:

harshdhami@localhost ~]$ sudo mkswap /dev/nvme0n2p4
mkswap: /dev/nvme0n2p4: warning: don't erase disk
Setting up swapspace version 1, size = 1 GiB
no label, UUID=[6497c71e-e752-4157-b397-21277da1e597]

harshdhami@localhost ~]$ sudo swapon /dev/nvme0n2p4

12. Create Logical Volumes:

# Initialize the physical volume
harshdhami@localhost ~]$ sudo pvcreate /dev/nvme0n2p5

# Create the volume group
harshdhami@localhost ~]$ sudo vgcreate VG1 /dev/nvme0n2p5

# Create logical volumes
harshdhami@localhost ~]$ sudo lvcreate -L 1G -n data1 VG1
harshdhami@localhost ~]$ sudo lvcreate -L 2G -n data2 VG1

13. Create an ext4 Filesystem on data1 LV:

harshdhami@localhost ~]$ sudo mkfs.ext4 /dev/VG1/data1

14. Create an ext4 Filesystem on data2 LV:

harshdhami@localhost ~]$ sudo mkfs.ext4 /dev/VG1/data2

22. Compression Commands:

# a. Compress a symbolic link
harshdhami@localhost ~]$ compress /root/sfile
harshdhami@localhost ~]$ compress -v /root/sfile

# b. Compress directory with gzip
harshdhami@localhost ~]$ gzip -r /root/dir1
harshdhami@localhost ~]$ gzip -v /root/dir1

# c. View contents of a zip file
harshdhami@localhost ~]$ unzip -l /root/letter.zip

# d. Compress a file with xz
harshdhami@localhost ~]$ xz -z /root/letter
harshdhami@localhost ~]$ xz -v /root/letter

# e. Find compression ratio of gzip file
harshdhami@localhost ~]$ gzip -l /root/letter.gz

# f. Test compression with bzip2
harshdhami@localhost ~]$ bzip2 -t /root/sample

# g. Compress with minimal memory usage
harshdhami@localhost ~]$ bzip2 --fast /root/sample

23. Backup Commands:

# 23. Backup /etc directory with tar
harshdhami@localhost ~]$ sudo tar cvf /root/etc.tar /etc

# 24. Append file to archive
harshdhami@localhost ~]$ sudo tar rvf /root/etc.tar /root/anaconda-ks.cfg

# 25. Create tarball of /home
harshdhami@localhost ~]$ sudo tar czvf /root/stuff.tar.gz /home

# 26. Backup with cpio
harshdhami@localhost ~]$ sudo find /etc | cpio -ov > /root/etc.cpio

# 27. Copy /etc to /dev/nvme0n2p1
harshdhami@localhost ~]$ sudo cp -a /etc /partition1

# 28. Full filesystem backup with dump
harshdhami@localhost ~]$ sudo dump -0uf /root/sdb1.dump /dev/nvme0n2p1
harshdhami@localhost ~]$ sudo dumpdate > /etc/dumpdates
